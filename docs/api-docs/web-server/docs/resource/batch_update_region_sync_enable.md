### 描述

- 该接口提供版本：v9.9.9+。
- 该接口所需权限：资源管理。
- 该接口功能描述：批量更新地域的同步启用状态。

### URL

PATCH /api/v1/cloud/vendors/{vendor}/regions/sync_enable/batch

#### 路径参数说明

| 参数名称   | 参数类型   | 必选  | 描述                                  |
|--------|--------|-----|-------------------------------------|
| vendor | string | 是   | 云厂商（当前仅支持 aws）                      |

### 输入参数

| 参数名称        | 参数类型     | 必选  | 描述                        |
|-------------|----------|-----|---------------------------|
| ids         | []string | 是   | 地域的数据库ID列表，最多100个         |
| sync_enable | bool     | 是   | 是否启用同步（true: 启用，false: 禁用） |

### 调用示例

#### 请求示例

禁用指定地域的同步功能：

```json
{
  "ids": ["0000000e", "0000000f", "00000010"],
  "sync_enable": false
}
```

启用指定地域的同步功能：

```json
{
  "ids": ["0000000e", "0000000f"],
  "sync_enable": true
}
```

### 响应示例

#### 成功响应

```json
{
  "code": 0,
  "message": "ok"
}
```

#### 失败响应

##### 参数校验失败

```json
{
  "code": 2000001,
  "message": "ids count should <= 100"
}
```

##### 地域不存在

```json
{
  "code": 2000001,
  "message": "some region(ids=[00000099]) doesn't exist"
}
```

##### 不支持的云厂商

```json
{
  "code": 2000000,
  "message": "vendor: tcloud not support"
}
```

### 响应参数说明

#### 通用返回响应说明

| 参数名称    | 参数类型   | 描述   |
|---------|--------|------|
| code    | int32  | 状态码  |
| message | string | 请求信息 |

### 错误码说明

| 错误码     | 描述        |
|---------|-----------|
| 0       | 成功        |
| 2000001 | 参数校验失败（InvalidParameter）    |
| 2000000 | 未知错误/不支持的云厂商（Unknown）   |

### 使用说明

1. **适用场景**：当需要控制特定地域是否参与资源同步时，可使用此接口批量启用或禁用地域的同步状态。
2. **影响范围**：禁用地域同步后，该地域下的云资源（如 CVM、VPC、子网等）将不会被同步到系统中。
3. **当前支持**：该接口当前仅支持 AWS 云厂商，其他云厂商暂不支持。
4. **批量限制**：单次请求最多更新 100 个地域。
5. **默认行为**：新同步的地域默认启用同步（sync_enable = true）。
